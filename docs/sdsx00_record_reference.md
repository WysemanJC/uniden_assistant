# SDSx00 File Specification — Parser Reference (Condensed)

Source: SDSx00_File_Specification_V1_03.pdf (converted to text for this document).

## 1) Global Parsing Rules

- **Encoding**: ASCII.
- **Line format**: One line = one sentence/record.
- **Field delimiter**: **Tab** (`\t`) separates command and parameters.
- **Sub-delimiters** inside fields: `=`, `/`, or `,` are used by the spec.

### MyId/ParentId: HPDB Resource Hierarchy

Each resource in HPDB has **two essential IDs** forming a parent-child relationship:

- **MyId** (self ID): Uniquely identifies the current resource within its category.
  - Format: `XxxId=xx` where `Xxx` is the resource type (e.g., `AgencyId=15`)
  - Scope: Unique within the parent container
  - Generated by: Device during import; parsed from HPDB files

- **ParentId** (parent ID): Identifies the resource's container/parent in the hierarchy.
  - Format: `XxxId=xx` matching the parent resource type
  - Establishes: Parent-child relationship for data organization
  - Null/empty: Allowed in Favorites Lists (spec note)

**HPDB Resource Hierarchy Chain:**
```
Country (CountryId=xx)
  └─ State (StateId=xx, ParentId=CountryId)
      └─ County (CountyId=xx, ParentId=StateId)
          └─ Agency (AgencyId=xx, ParentId=CountyId)
              ├─ Conventional Frequency (CFreqId=xx, ParentId=AgencyId)
              ├─ Channel Group (CGroupId=xx, ParentId=AgencyId)
              │   └─ Conventional Frequency (CFreqId=xx, ParentId=CGroupId)
              └─ [LM/TGID data]
          └─ Site (SiteId=xx, ParentId=TrunkId)
              ├─ Trunking System (TrunkId=xx, ParentId=CountyId)
              │   ├─ Talkgroup (TGroupId=xx, ParentId=TrunkId)
              │   │   └─ Trunking Frequency (TFreqId=xx, ParentId=TGroupId)
              │   └─ Talkgroup Frequency (TFreqId=xx, ParentId=SiteId)
```

**Common ID Parameters:**
- `CountryId`: Country code (e.g., `CountryId=1` for USA, `CountryId=2` for Canada)
- `StateId`: State within country (e.g., `StateId=06` for California)
- `CountyId`: County within state
- `AgencyId`: Agency/system within county
- `TrunkId`: Trunking system identifier
- `SiteId`: Individual site/repeater identifier
- `CGroupId`: Conventional channel group
- `CFreqId`: Conventional frequency record
- `TGroupId`: Talkgroup (trunking group)
- `TFreqId`: Trunking frequency record
- `Tid`: Talkgroup ID (numeric)

**Special Cases:**
- **Favorites List `.hpd` files**: MyId and ParentId may be **null** or empty (spec note)
- **Scan Configuration records** (`ScanListType`, `ScanConvSystem`, etc.): Use ParentId to reference parent list/system
- **Area records** (`AreaState`, `AreaCounty`): Link geographic coverage using parent IDs

- **Name Tag**: ASCII 0x20–0x7E, max length 64.
- **Frequency**: stored as Hz integers; 1 = 1Hz (e.g., 851.0125MHz → 851012500).
- **Latitude/Longitude**: degree format; up to 6 decimals.
- **Timestamp**: `MM/DD/YYYY hh:mm:ss`.
- **Avoid in HPDB**: typically `Off`; actual avoid handled in `*.avoid`.

### TGID (Talkgroup ID) Format Specifications

TGID formats vary by system type. Below are all supported formats:

#### **MOTOROLA TYPE I**
Format validation: Varies by pattern (see Types 1-7 below)

1. **BFF-SS** (Normal ID)
   - B: 1 digit (BLOCK)
   - FF: 2 digits, 0-filled (FLEET)
   - SS: 1-2 digits (SUB-FLEET)
   - Example: `1-02-5`, `2-15-12`

2. **BFFF-S** (Normal ID when FLEET is 100-127)
   - B: 1 digit (BLOCK)
   - FFF: 3 digits (FLEET)
   - S: 1 digit (SUB-FLEET)
   - Example: `1-100-5`, `2-127-9`

3. **B-** (Partial ID: free FLEET and SUB-FLEET)
   - B: 1 digit (BLOCK)
   - Wildcard: represents any FLEET/SUB-FLEET

4. **BFF-** (Partial ID: free SUB-FLEET)
   - B: 1 digit (BLOCK)
   - FF: 2 digits, 0-filled (FLEET)
   - Wildcard: represents any SUB-FLEET

5. **BFFF-** (Partial ID when FLEET is 100-127)
   - B: 1 digit (BLOCK)
   - FFF: 3 digits (FLEET)
   - Wildcard: represents any SUB-FLEET

6. **NNNNN** (Normal ID when SIZE CODE is 0)
   - NNNNN: 1-5 digits (numeric ID)

7. **iNNNNN** (I-CALL ID)
   - i: prefix (literal 'i')
   - NNNNN: 1-5 digits (numeric ID)

**Note**: TYPE I format has no format validation.

#### **MOTOROLA TYPE II**
Format validation: Strict numeric with optional i-prefix

1. **NNNNN** (Normal ID)
   - NNNNN: 1-5 digits
   - Range: 1–65535

2. **iNNNNN** (I-CALL ID)
   - i: prefix (literal 'i')
   - NNNNN: 1-5 digits
   - Range: i0–i65535

#### **P25**
Format validation: Strict numeric with optional i-prefix

1. **NNNNN** (Normal ID)
   - NNNNN: 1-5 digits
   - Range: 1–65535

2. **iNNNNNNNN** (I-CALL ID)
   - i: prefix (literal 'i')
   - NNNNNNNN: 1-8 digits
   - Range: i0–i16777215

**Note**: P25 format does NOT support Partial IDs.

#### **DMR / MotoTRBO**
Format validation: Strict numeric with optional i-prefix

1. **NNNNNNNN** (Normal ID)
   - NNNNNNNN: 1-8 digits
   - Range: 1–16777215

2. **iNNNNNNNN** (I-CALL ID)
   - i: prefix (literal 'i')
   - NNNNNNNN: 1-8 digits
   - Range: i0–i16777215

**Note**: DMR format does NOT support Partial IDs.

#### **NXDN**
Format validation: Strict numeric with optional i-prefix

1. **NNNNN** (Normal ID)
   - NNNNN: 1-5 digits
   - Range: 0–65535

2. **iNNNNN** (I-CALL ID)
   - i: prefix (literal 'i')
   - NNNNN: 1-5 digits
   - Range: i0–i65535

**Note**: NXDN does NOT support Partial IDs.

#### **EDACS**
Format validation: Varies by pattern

1. **AA-FFS** (Normal ID)
   - AA: 2 digits, 0-filled (AGENCY: 00–15)
   - FF: 2 digits, 0-filled (FLEET: 00–15)
   - S: 1 digit (SUB-FLEET: 0–7)
   - Range: decimal 1–2047
   - Example: `00-00-1`, `15-15-7`

2. **AA---** (Partial ID: free FLEET and SUB-FLEET)
   - AA: 2 digits, 0-filled (AGENCY)
   - Wildcard: represents any FLEET/SUB-FLEET

3. **AA-FF-** (Partial ID: free SUB-FLEET)
   - AA: 2 digits, 0-filled (AGENCY)
   - FF: 2 digits, 0-filled (FLEET)
   - Wildcard: represents any SUB-FLEET

4. **iNNNNNN** (I-CALL ID)
   - i: prefix (literal 'i')
   - NNNNNN: 1-7 digits
   - Range: i0–i1048575

5. **NNNNN** (EA: Extend Addressing System)
   - NNNNN: 1-5 digits
   - Range: 2048–65535
   - Note: Special mode for extended addressing

**Constraint**: `00-00-0` (all zeros) is invalid.

#### **LTR**
Format validation: Fixed pattern

1. **A-RR-NNN** (Normal ID)
   - A: 1 digit (AREA CODE: 0–1)
   - RR: 2 digits, 0-filled (HOME REPEATER: 01–20)
   - NNN: 3 digits, 0-filled (ID: 000–254)
   - Example: `0-01-000`, `1-20-254`

2. **A-RR---** (Partial ID)
   - A: 1 digit (AREA CODE)
   - RR: 2 digits, 0-filled (HOME REPEATER)
   - Wildcard: represents any ID

**Note**: LTR system does NOT support I-CALL IDs.

### Common Parameter Types

This section defines frequently used parameter types referenced throughout the specification.

#### **Alert Tone Type**
Specifies the tone to play for alerts on a frequency or talkgroup.
- `Off`: No tone
- `1` – `9`: Specific tone IDs (selectable alert tones)

#### **Alert Tone Volume**
Specifies the volume level for alert tones.
- `Auto`: Device automatically adjusts volume based on current setting
- `1` – `15`: Manual volume level (1 = minimum, 15 = maximum)

#### **Alert Light Color**
Specifies the LED light color for alerts.
- `Off`: LED disabled for this item
- `Blue`: Blue LED
- `Red`: Red LED
- `Magenta`: Magenta LED
- `Green`: Green LED
- `Cyan`: Cyan LED
- `Yellow`: Yellow LED
- `White`: White LED

#### **Alert Light Pattern**
Specifies the LED light animation pattern for alerts.
- `On`: LED steady on (no blinking)
- `Slow Blink`: LED blinks slowly
- `Fast Blink`: LED blinks rapidly

#### **Digital Waiting Time**
Time (in milliseconds) that the receiver waits for digital signal after detecting carrier.
- Range: `0–1000` milliseconds
- Unit: 1 = 1 millisecond
- Common values: 0 (immediate), 100 (0.1s), 500 (0.5s), 1000 (1.0s)

#### **Digital Threshold Mode**
Specifies how the receiver determines digital signal presence.
- `Auto`: Automatically adjusts threshold based on signal strength
- `Manual`: Uses fixed threshold set by Digital Threshold Level
- `Default`: Uses device default threshold

#### **Digital Threshold Level**
Sets the signal strength threshold for detecting digital signals (used when mode is Manual).
- Range: `5–13` (integer)
- Lower values = more sensitive
- Higher values = less sensitive (requires stronger signal)
- Note: Only used when Digital Threshold Mode is set to `Manual`

### Display Options & Area Sizing

Display area options define how UI information is rendered on the device. Options are split into 4 **DispOptId** groups, each supporting different display layouts.

#### **Display Area Size Options (DispOptId)**

Each DispOptId maps to a display area size category:

- **DispOptId=1**: `Huge area option` — Full-screen, maximum visibility
- **DispOptId=2**: `Large area option` — 50–75% screen space
- **DispOptId=3**: `Small area option` — 25–50% screen space
- **DispOptId=4**: `Icon area option` — Icon-only display (≤25% screen space, 10 icon positions)

#### **Display Layout Modes (DispLayoutId & ColorLayoutId)**

Display modes determine which DispOptItems and DispColors are used:

| DispLayoutId | ColorLayoutId | Mode | Items | Colors |
|---|---|---|---|---|
| 1 | 1 | Simple Conventional | DispOptId=1,2,3,4 | DispColorId=1–7 |
| 2 | 6 | Simple Trunk | DispOptId=1,2,3,4 | DispColorId=1–7 |

| 4 | 7 | Detail Trunk | DispOptId=1,2,3,4 | DispColorId=1–7 |

| 7 | 5 | Tone Out | DispOptId=2,3,4 | DispColorId=1–7 |

#### **Display Color Codes (DispColorId)**

Colors are stored as hex RGB triplets (TextColor / BackColor pairs). Each DispColorId maps to a specific UI element or layout component:

- **DispColorId=1**: System/header information (System Name, Department Name, Channel Name, Info areas)
- **DispColorId=2**: Primary options (System Option, Dept Option, Channel Option)
- **DispColorId=3**: Secondary options (Option_1–Option_8)
- **DispColorId=4**: Extended options (Option A/B sets)
- **DispColorId=5**: Icon display colors (ICON1–ICON10)
- **DispColorId=6**: Status bar (F, SIG, BAT, KEY, DIR buttons)
- **DispColorId=7**: Soft keys (Soft Key 1–3, SP0–SP2)

**Color Reversal Note**: F, Soft Key 1, Soft Key 2, and Soft Key 3 use reversed text/back colors (swap for UI emphasis).

### Channel/Frequency Options by System Type — Complete Reference

This section provides comprehensive mappings for all frequency/channel options across system types. Each subsection includes Display format, File format, and Discovery folder naming conventions.

**Navigation:**
- [Conventional Channel Options (CTCSS/DCS)](#conventional-channel-options-ctcssdcs)
- [P25 Frequency Options (NAC)](#p25-frequency-options-nac)
- [DMR Frequency Options (Color Code)](#dmr-frequency-options-color-code)
- [NXDN Frequency Options (RAN/Area)](#nxdn-frequency-options-ranarea)

**Key Discovery Rules:**
- **Analog systems**: If no CTCSS/DCS is specified, Discovery shows "None"
- **Digital systems**: NAC or RAN or Area code always exists in Discovery
- **Tone L/O**: ",L/O" suffix is appended when Tone Lockout is enabled (e.g., "CTCSS 67.0Hz,L/O")

## 3) File/Folder Layout (Top Level)

Key directories and files (abbrev):
- `scanner.inf` — scanner information (`Scanner` record)
- `profile.cfg` — profile/settings (`ProductName`, `GlobalSetting`, etc.)
- `app_data.cfg` — runtime scan state (`Scan*` records, `ModeInfo`, `ScanListType`)
- `favorites_lists/f_list.cfg` — favorites list index (`F-List` records)
- `favorites_lists/f_*.hpd` — favorites list systems (same record types as HPDB with null IDs in some cases)
- `HPDB/hpdb.cfg` — HPDB metadata (`DateModified`, `StateInfo`, `CountyInfo`, `LM`, `LM_Frequency`)
- `HPDB/s_*.hpd` — HPDB systems and channels (records in section 4)
- `*.avoid` — avoid lists (`Avoid` records)
- `discovery.cfg` — discovery settings (`ConvDiscovery`, `TrunkDiscovery`)

## 4) HPDB Records (hpdb.cfg)

### [DateModified]
**Format**: `DateModified\tTimeStamp`
- TimeStamp: `MM/DD/YYYY hh:mm:ss`

### [StateInfo]
**Format**: `StateInfo\tStateId\tCountryId\tNameTag\tShortName`
- StateId: `StateId=xx` (integer)
- CountryId: `CountryId=xx` (integer)
- NameTag: ASCII string, max 64 chars
- ShortName: 2-letter code (e.g., AL, TX)

### [CountyInfo]
**Format**: `CountyInfo\tCountyId\tStateId\tNameTag`
- CountyId: `CountyId=xx` (integer)
- StateId: `StateId=xx` (integer)
- NameTag: ASCII string, max 64 chars

### [LM]
**Format**: `LM\tStateId\tCountyId\tTrunkId\tSiteId\tLM_SystemID\tLM_SiteID\tLatitude\tLongitude`
- StateId: `StateId=xx` (integer)
- CountyId: `CountyId=xx` (integer)
- TrunkId: `TrunkId=xx` (integer)
- SiteId: `SiteId=xx` (integer)
- LM_SystemID: string (Locate Me system identifier)
- LM_SiteID: string (Locate Me site identifier)
- Latitude: degree format, up to 6 decimals
- Longitude: degree format, up to 6 decimals
- **Note**: First sort key = LM_SystemID; second sort key = LM_SiteID

### [LM_Frequency]
**Format**: `LM_Frequency\tFrequency\tReserve\tLmTypeArray`
- Frequency: Hz integer (1 = 1Hz)
- Reserve: (reserved field)
- LmTypeArray: bit flags encoding detected system types
  - Bit 0 (LSB): Motorola (1 = detected, 0 = not detected)
  - Bit 1: P25 Standard (1 = detected, 0 = not detected)
  - Bit 2: P25 X2-TDMA (1 = detected, 0 = not detected)

## 5) HPDB/System Records (s_*.hpd)

### TargetModel
**Format**: `TargetModel\tModelName`

### FormatVersion
**Format**: `FormatVersion\tVersion`

### Conventional
**Format**: `Conventional\tMyId\tParentId\tNameTag\tAvoid\tReserve\tSystemType\tQuickKey\tNumberTag\tSystemHoldTime\tAnalogAGC\tDigitalAGC\tDigitalWaitingTime\tDigitalThresholdMode\tDigitalThresholdLevel`
- MyId: `CountyId=xx` or `AgencyId=xx`
- ParentId: `StateId=xx`

### Trunk
**Format**: `Trunk\tMyId\tParentId\tNameTag\tAvoid\tReserve\tSystemType\tIDSearch\tAlertTone\tAlertVolume\tStatusBit\tNAC\tQuickKey\tNumberTag\tSiteHoldTime\tAnalogAGC\tDigitalAGC\tEndCode\tPriorityIDScan\tAlertColor\tAlertPattern\tTGIDFormat`
- SystemType: Conventional, Motorola, Edacs, Scat, Ltr, P25Standard, P25OneFrequency, P25X2_TDMA, MotoTrbo, DmrOneFrequency, Nxdn, NxdnOneFrequency

### AreaState
**Format**: `AreaState\tMyId\tStateId`
- MyId: `CountyId=xx` / `AgencyId=xx` / `TrunkId=xx`

### AreaCounty
**Format**: `AreaCounty\tMyId\tCountyId`
- MyId: `CountyId=xx` / `AgencyId=xx` / `TrunkId=xx`

### FleetMap
**Format**: `FleetMap\tMyId\tB0\tB1\tB2\tB3\tB4\tB5\tB6\tB7`
- Bx: `0` or `1-9/A/B/C/D/E` (block sizes)

### UnitIds
**Format**: `UnitIds\tReserve\tReserve\tNameTag\tUnitId\tAlertTone\tAlertVolume\tAlertColor\tAlertPattern`

### AvoidTgids
**Format**: `AvoidTgids\tMyId\tTGID1\tTGID2\t...\tTGID16`
- Multiple lines if >16 TGIDs

### Site
**Format**: `Site\tMyId\tParentId\tNameTag\tAvoid\tLatitude\tLongitude\tRange\tModulation\tMotBandType\tEdacsBandType\tLocationType\tAttenuator\tDigitalWaitingTime\tDigitalThresholdMode\tDigitalThresholdLevel\tQuickKey\tNAC\tFilter`
- LocationType: Circle/Rectangles
- MotBandType: Standard/Sprinter/Custom
- EdacsBandType: Wide/Narrow

### Rectangle
**Format**: `Rectangle\tMyId\tLatitude1\tLongitude1\tLatitude2\tLongitude2`
- MyId: `SiteId=xx`, `TGroupId=xx`, or `CGroupId=xx`
- Note: “Rectangle sentence is additional information for Department or Site.”

### BandPlan_Mot
**Format**: `BandPlan_Mot\tMyId\tLower0\tUpper0\tSpacing0\tOffset0\t...\tLower5\tUpper5\tSpacing5\tOffset5`
- Valid only when MotBandType is Custom

### BandPlan_P25
**Format**: `BandPlan_P25\tMyId\tBase0\tSpacing0\tBase1\tSpacing1\t...\tBaseF\tSpacingF`
- Optional if band plan is undesignated

### [DQKs_Status]
- MyId: (parent context)
- D-Qkey_00–D-Qkey_99: `Off` | `On` (boolean, one per quick key)

### [C-Group]
- MyId: `CGroupId=xx` (integer)
- ParentId: `AgencyId=xx` (integer)
- NameTag: ASCII string, max 64 chars
- Avoid: `On` | `Off` (always `Off` in HPDB)
- Latitude: degree format, up to 6 decimals
- Longitude: degree format, up to 6 decimals
- Range: `1=1mile`, up to 1 decimal point
- LocationType: `Circle` | `Rectangles` (enum)
- QuickKey: `Off` | `0-99` (integer)
- Filter: `Global` | `Normal` | `Invert` | `Auto` (enum)

### [T-Group]
- MyId: `TGroupId=xx` (integer)
- ParentId: `TrunkId=xx` (integer)
- NameTag: ASCII string, max 64 chars
- Avoid: `On` | `Off` (always `Off` in HPDB)
- Latitude: degree format, up to 6 decimals
- Longitude: degree format, up to 6 decimals
- Range: `1=1mile`, up to 1 decimal point
- LocationType: `Circle` | `Rectangles` (enum)
- QuickKey: `Off` | `0-99` (integer)

### [C-Freq]
- MyId: `CFreqId=xx` (integer)
- ParentId: `CGroupId=xx` (integer)
- NameTag: ASCII string, max 64 chars
- Avoid: `On` | `Off` (always `Off` in HPDB)
- Frequency: Hz integer (1 = 1Hz)
- Modulation: `AUTO` | `AM` | `NFM` | `FM` (enum)
- AudioOption: `null` (all) | `Tone=xxxx` | `NAC=xxx` | `ColorCode=xx` | `RAN=xx` | `Area=x` (see Sub Audio sheet)
- FuncTagId: `1-37` | `208-218` (integer; see Service Type sheet)
